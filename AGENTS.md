# AGENTS.md

## Project Overview

Personal tech blog for Software Engineer Antonio Barbosa (aboneto). Currently in **mockup phase** — `mockup/` contains a static design canvas with terminal/dev aesthetic. Jekyll site not yet initialized.

**Stack (planned):** Jekyll, SASS, OpenSpec. High priority on **SEO and performance**.

## Image Workflow (Critical)

Las imágenes tienen un flujo obligatorio — no saltarse pasos:

1. **Raw images** (PNG, JPG) → depositar en `assets/images/`
2. **Ejecutar** `scripts/convert_to_webp.sh` para convertir a `.webp`
3. **Referenciar siempre** la salida `.webp` en `assets/img/`
4. **NUNCA** referenciar imágenes raw directamente en el código

## Commands

```bash
# Jekyll (una vez inicializado)
bundle exec jekyll serve    # dev server
bundle exec jekyll build    # build estático

# Conversión de imágenes
./scripts/convert_to_webp.sh
```

## Conventions

- **Idioma del chat:** Responder siempre en español
- **Commits:** Inglés, Conventional Commits (`feat:`, `style:`, `perf:`, `fix:`, `chore:`)
- **Ramas:** Inglés, `tipo/nombre-de-la-rama` (ej. `feat/nuevo-post`, `style/grid-css`)
- **Código:** En inglés (nombres de variables, funciones, archivos)
- **Posts:** En español
- **Sintaxis SCSS:** Usar siempre `.scss` con llaves `{}` y punto y coma `;`. NUNCA usar sintaxis indentada `.sass`

## Design System (from mockup)

- **Palette:** `--bg: #0a1530`, `--accent: #7aa2ff`, `--green: #7ad6a3`
- **Fonts:** IBM Plex Sans (body), IBM Plex Mono (terminal/meta)
- **Aesthetic:** Terminal/dev — `$` prompt, cursor parpadeante, frontmatter YAML visible, breadcrumbs tipo path

## SEO & Performance Notes

- Imágenes siempre en `.webp` (ver flujo arriba)
- Usar `loading="lazy"` en imágenes below-the-fold
- Mantener el CSS crítico inline cuando sea posible
- Structured data y meta tags completos en cada página

# RTK (Rust Token Killer) - Token-Optimized Commands

## Golden Rule

**Always prefix commands with `rtk`**. If RTK has a dedicated filter, it uses it. If not, it passes through unchanged. This means RTK is always safe to use.

**Important**: Even in command chains with `&&`, use `rtk`:
```bash
# ❌ Wrong
git add . && git commit -m "msg" && git push

# ✅ Correct
rtk git add . && rtk git commit -m "msg" && rtk git push
```
